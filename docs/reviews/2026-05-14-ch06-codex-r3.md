# 챕터 06 리뷰 — 라운드 3

## ACCEPT 1

- 위치: `docs/report/06-ui-and-board.md:51`, `docs/report/06-ui-and-board.md:64`
- 현재: `ui/src/adapters/`를 “어댑터별 동적 폼/뷰” 확장점으로 설명하면서 “각 어댑터의 `./ui` export를 dynamic import”한다고 적습니다.
- 문제: 현재 UI의 built-in adapter UI는 `registry.ts`에서 정적 import 후 등록됩니다. 외부 어댑터의 동적 부분도 각 패키지의 `./ui` export를 브라우저에서 `import()`하는 방식이 아니라, `/api/adapters/:type/ui-parser.js`를 fetch해서 sandboxed Web Worker 안에서 parser source를 실행하는 방식입니다. 반면 플러그인 UI는 `ui/src/plugins/slots.tsx`에서 dynamic ESM `import()`를 사용하므로, 어댑터 UI와 플러그인 UI의 로딩 계약을 구분해야 합니다.
- 제안: `adapters/` 설명을 “built-in 어댑터 UI registry + 외부 어댑터 `ui-parser.js` sandboxed parser bridge”로 바꾸고, dynamic ESM import 표현은 `plugins/` 설명에만 남기세요.
