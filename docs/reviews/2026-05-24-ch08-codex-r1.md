- **문제 카테고리**: 정합성
- **제안 수정안**: `후속 임원/에이전트도 현재 구현에서는 명시적인 agent hire 흐름으로 생성된다. \`require_board_approval_for_new_agents=true\`이면 각 hire가 \`hire_agent\` approval로 들어가고, 보드 승인 후 활성화된다.`처럼 현재 구현 기준으로 정정.
- **근거**: 구현된 신규 에이전트 승인 게이트는 `POST /companies/:companyId/agent-hires`에서 `requireBoardApprovalForNewAgents`일 때 `agents.status='pending_approval'`와 `approvals.type='hire_agent'`를 만드는 흐름이다(`server/src/routes/agents.ts:1945-2008`, `:2014-2062`). 승인 처리도 `hire_agent`일 때 pending agent를 활성화하는 분기만 실제 side effect를 갖는다(`server/src/services/approvals.ts:102-168`). `approve_ceo_strategy`는 상수로만 존재한다(`packages/shared/src/constants.ts:408-413`).
