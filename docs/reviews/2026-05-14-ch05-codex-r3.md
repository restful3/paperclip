## ACCEPT-worthy

### R3-A1. 위치: `05-server-api.md:129-142`

- 현재: "Better Auth + 세 종류의 키"라고 제목을 달고, `server/src/auth/`가 Better Auth 위에 세 가지 인증 주체를 얹는다고 설명한다. 표에는 보드 사용자, local-board, agent API key, board API key 네 행이 있고, agent credential은 `agent_api_keys`만 언급한다.
- 문제: 현재 REST actor middleware의 credential 경로는 session/local implicit board, board API key, persistent agent API key, local agent JWT까지 포함한다. 특히 local adapters에는 heartbeat 실행 시 `createLocalAgentJwt`가 발급되고, `actorMiddleware`는 persistent key 조회 실패 시 `verifyLocalAgentJwt`로 `source: "agent_jwt"` actor를 만든다. 따라서 "세 종류의 키"와 agent API key 단독 설명은 현재 인증 계약을 빠뜨린다.
- 제안: 제목/도입을 "Better Auth + board/agent credentials" 또는 "Better Auth + 세 가지 bearer 경로"처럼 바꾸고, 표의 에이전트 행을 "agent API key 또는 local agent JWT"로 확장한다. 필요하면 board 사용자/local-board는 credential이 아니라 board actor mode로 분리한다.
- 근거: `server/src/middleware/auth.ts:24-34`, `server/src/middleware/auth.ts:109-129`, `server/src/middleware/auth.ts:131-199`, `server/src/agent-auth-jwt.ts:68-95`, `server/src/services/heartbeat.ts:7649`.
