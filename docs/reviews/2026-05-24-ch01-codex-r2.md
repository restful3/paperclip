### 2. PR 전 검증 라인 표기가 표 2의 명령명과 어긋남

- **위치**: line 97
- **현재 본문 인용**: ``test`/`test:e2e`/`-r typecheck`/`-r build` 가 PR 전 검증 라인이다.`
- **문제 카테고리**: 품질
- **제안 수정안**: ``test`/`test:e2e`/`typecheck`/`build` 가 PR 전 검증 라인이다.`
- **근거**: 바로 위 표 2는 `pnpm typecheck`와 `pnpm build`를 표준 명령으로 제시한다. 실제 root script도 `package.json:16-17`에서 `build`와 `typecheck`가 각각 preflight 후 `pnpm -r build/typecheck`를 실행한다. `-r typecheck`/`-r build`라고만 쓰면 직접 recursive 명령을 뜻하는지 표준 wrapper 명령을 뜻하는지 흐려진다.
