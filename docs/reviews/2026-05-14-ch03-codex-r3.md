# 챕터 03 리뷰 — 라운드 3

## 종합 판정

R2의 세 수정은 대부분 반영됐다. recovery action 유니크 키 설명, 7단계 startup reconciliation 제목과 순서, wildcard 전이 bullet은 정리됐다. 다만 `cancelled` 설명에 전이 범위를 다시 과장하는 문장이 남아 있어 ACCEPT-worthy로 본다.

## ACCEPT-worthy

### R3-A1. 위치: `03-runtime-execution.md:68`

- 현재: "`cancelled` 가 흡수 상태라서 **어떤 상태에서든** 이쪽으로 들어오면 다시 나가지 않는다"고 설명.
- 문제: V1 전이표는 `done -> cancelled` 를 허용하지 않는다. `cancelled` 가 terminal인 것은 맞지만, "어떤 상태에서든 진입"은 R2에서 제거한 wildcard 전이와 같은 오해를 다시 만든다.
- 제안: "`cancelled` 는 terminal state라서 일단 진입하면 다시 나가지 않는다. 단, 진입은 V1 전이표가 명시한 비종료 상태에서만 다룬다"로 수정.
- 근거: `doc/SPEC-implementation.md:421-426`.
