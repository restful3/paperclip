# ch05 (05-server-api.md) Round 3 리뷰 — Codex

## 1. companies 대표 PATCH 경로의 파라미터명이 실제 라우트와 다름

**위치**  
`docs/report/05-server-api.md` §2 표 1 `companies.ts` 행

**현재 본문 인용**  
> `companies.ts` | 회사 lifecycle | `GET/POST /api/companies`, `PATCH /api/companies/:id`

**문제 카테고리**  
정합성 — 엄격

**제안 수정안**  
> `companies.ts` | 회사 lifecycle | `GET/POST /api/companies`, `PATCH /api/companies/:companyId`

**근거**  
`companyRoutes`는 `app.ts`에서 `/api/companies` 아래에 mount되고, PATCH 라우트 파라미터명은 `:companyId`입니다.

```ts
// server/src/app.ts
208  api.use("/companies", companyRoutes(db, opts.storageService));
```

```ts
// server/src/routes/companies.ts
329  router.patch("/:companyId", async (req, res) => {
330    const companyId = req.params.companyId as string;
331    assertCompanyAccess(req, companyId);
```

같은 표가 Round 2에서 `projects.ts`의 `:pid` → `:id`를 엄격하게 정정했으므로, 여기서도 실제 라우트 파라미터명에 맞추는 편이 일관됩니다.
