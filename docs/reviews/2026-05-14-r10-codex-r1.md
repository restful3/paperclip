공식 가격표와 Paperclip 비용 스키마의 핵심 매핑은 대체로 맞습니다. 다만 Gemini context caching 의 storage 단가가 빠져 있어 "정확한 재현" 주장과 충돌하고, LangSmith 비용 산정 설명은 현재 붙은 출처보다 전용 Cost tracking 문서를 인용해야 합니다.

### 1. Gemini context caching storage 비용 누락

**위치**: L15

**현재**: Google 의 **Gemini 2.5 Pro**는 입력 길이로 가격을 분기시키는 독특한 구조다. 200k 토큰 이하는 input $1.25 / cached $0.125 / output $10이고, 200k 초과 프롬프트는 input $2.50 / cached $0.25 / output $15로 두 배 단가가 적용된다.

**문제**: [ACCEPT] 사실 정확성 위반. Gemini 2.5 Pro Standard tier 의 per-token 수치는 맞지만, 공식 가격표는 context caching 에 대해 token 단가와 별도로 storage price `$4.50 / 1,000,000 tokens per hour` 를 함께 둔다(https://ai.google.dev/gemini-api/docs/pricing, Gemini 2.5 Pro Standard 표). 본문은 "청구가 정확히 재현"되어야 한다고 결론 내리므로, Gemini의 시간 기반 cache storage 축을 빼면 가격 구조 설명이 불완전하다.

**제안**: Gemini 문장에 "context caching storage 는 별도로 $4.50 / 1M tokens / hour"를 추가하고, 결론도 "모델·공급자·캐시 종류·캐시 storage 시간"처럼 시간 기반 modifier를 포함하도록 고친다.

### 2. LangSmith 비용 산정 출처

**위치**: L25, L51

**현재**: **LangSmith**는 trace tree 기반으로 inputs/outputs/metadata를 기록하지만 비용은 모델별 단가 매핑을 통해 사후 산정한다 ([LangSmith Observability Concepts](https://docs.langchain.com/langsmith/observability-concepts)).

**문제**: [ACCEPT] 인용 완결성 위반. `observability-concepts` 문서는 trace/project/run/metadata 구조와 trace 기록 개념을 설명하지만, 모델별 단가 매핑으로 cost 를 계산한다는 세부는 전용 `Cost tracking` 문서가 직접 설명한다. 해당 문서는 token counts, model/provider, model price 를 기반으로 자동 cost 를 계산하고 model pricing map 으로 비용을 산정한다고 명시한다(https://docs.langchain.com/langsmith/cost-tracking).

**제안**: 본문 인용을 `https://docs.langchain.com/langsmith/cost-tracking` 으로 바꾸고, §6 출처에도 `LangSmith — Cost tracking` 을 추가하거나 `Observability Concepts` 와 분리한다.
