1. [8-15] §4 회사 설정 주석이 월 예산 위치를 잘못 묶었습니다.
   - 본문 인용: "`description` · `brand_color` · 월 예산(`budget_monthly_cents`) · `attachment_max_bytes` 등은 회사 설정(`/company/settings`)에서 별도로 채운다."
   - 진실 소스: `ui/src/pages/CompanySettings.tsx:252`, `ui/src/pages/CompanySettings.tsx:333`, `ui/src/pages/CompanySettings.tsx:367`, `server/src/routes/costs.ts:280`, `ui/src/App.tsx:125`
   - 분류: ❌ 사실 오류
   - 설명: `/company/settings`에는 description, brand color, attachment size limit 등이 있지만 회사 월 예산 변경은 cost/budget API(`PATCH /companies/:companyId/budgets`)와 비용 화면(`/costs`) 쪽입니다. "월 예산은 비용/예산 화면에서 조정"처럼 분리하는 편이 정확합니다.

2. [8-16] §8 단독 Docker 실행 예시에서 필수 `BETTER_AUTH_SECRET`이 빠졌습니다.
   - 본문 인용: "`docker run -p 3100:3100 -v paperclip-data:/paperclip paperclip:dev`"
   - 진실 소스: `Dockerfile:76`, `server/src/auth/better-auth.ts:95`, `server/src/auth/better-auth.ts:98`, `doc/DOCKER.md:31`, `doc/DOCKER.md:35`
   - 분류: ❌ 사실 오류
   - 설명: 루트 Dockerfile은 기본 `PAPERCLIP_DEPLOYMENT_MODE=authenticated`이므로 Better Auth secret 없이는 서버가 시작될 수 없습니다. `doc/DOCKER.md`의 one-liner처럼 `-e BETTER_AUTH_SECRET=$(openssl rand -hex 32)`를 단독 컨테이너 예시에도 넣어야 합니다.

3. [8-17] §9 authenticated/public 환경 변수 묶음에 모드/노출 설정이 빠져 있습니다.
   - 본문 인용: "# authenticated/public 모드 필수"
   - 진실 소스: `server/src/config.ts:160`, `server/src/config.ts:165`, `server/src/config.ts:171`, `server/src/config.ts:177`, `cli/src/checks/deployment-auth-check.ts:51`
   - 분류: ❌ 사실 오류
   - 설명: env-only 호스팅에서 `authenticated/public` 자체를 만들려면 `PAPERCLIP_DEPLOYMENT_MODE=authenticated`와 `PAPERCLIP_DEPLOYMENT_EXPOSURE=public`도 필요합니다. 현재 블록은 auth secret/base URL만 보여 주어, 기본값(`local_trusted` 또는 private exposure)과 혼동될 수 있습니다.

4. [8-18] §9 board-claim 설명이 조건부 흐름을 일반화합니다.

   - 진실 소스: `doc/DEPLOYMENT-MODES.md:115`, `doc/DEPLOYMENT-MODES.md:117`, `server/src/board-claim.ts:52`, `server/src/board-claim.ts:57`, `server/src/board-claim.ts:58`
