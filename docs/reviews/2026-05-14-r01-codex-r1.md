R01의 큰 분류 체계와 Paperclip을 control plane으로 놓는 결론은 대체로 맞습니다. 다만 직접 인용으로 처리한 외부 문장 몇 곳이 원문과 일치하지 않고, Paperclip의 budget hard-stop 범위가 현재 코드의 집행 범위보다 넓게 쓰여 수정이 필요합니다.

### 1. Anthropic workflow/agent 직접 인용 불일치

**위치**: L21

**현재**: `"Workflows: LLMs and tools are orchestrated through predefined code paths"; "Agents: systems where LLMs dynamically direct their own processes and tool usage."`

**문제**: [ACCEPT] 사실/인용 위반. Anthropic 원문은 `Workflows:` / `Agents:` 라벨형 문장이 아니고, 두 번째 문장도 현재 본문처럼 축약된 quote가 아닙니다. 직접 quote로 둘 거면 원문 문장 구조와 생략 여부가 맞아야 합니다. 출처: https://www.anthropic.com/research/building-effective-agents

**제안**: 직접 인용을 원문 그대로 고치거나, 따옴표를 제거해 “Anthropic은 workflows를 사전 정의된 code path, agents를 LLM이 process와 tool use를 동적으로 지시하는 시스템으로 구분한다”처럼 paraphrase 처리하세요.

### 2. Temporal Event History quote 변형

**위치**: L25

**현재**: `"Temporal records a full Event History — every single time code in the Workflow is run, every single time an Activity is called or returned."`

**문제**: [ACCEPT] 사실/인용 위반. Temporal 블로그의 해당 문장은 현재 본문처럼 시작하지 않고, Activity 호출/반환 뒤의 추가 범위도 남아 있습니다. 현재 문장은 의미는 가깝지만 직접 quote로는 원문과 다릅니다. 출처: https://temporal.io/blog/durable-execution-meets-ai-why-temporal-is-the-perfect-foundation-for-ai

**제안**: 따옴표를 제거하고 “Temporal은 Workflow 실행과 Activity 호출/반환을 Event History에 기록해 재시도와 복구의 기반으로 삼는다고 설명한다”로 바꾸거나, 원문 문장을 정확히 인용하세요.

### 3. LangGraph quote 원문 부재

**위치**: L33

**현재**: `"agent never loses progress across restarts" — checkpoint + thread_id`

**문제**: [ACCEPT] 사실/인용 위반. LangGraph durable execution 문서는 workflow/process의 진행 상태 저장과 “last recorded state” 재개를 설명하지만, 현재 quote 문구는 원문에서 확인되지 않습니다. 또한 문서의 대상은 일반적인 agent 전체라기보다 durable workflow 실행입니다. 출처: https://docs.langchain.com/oss/python/langgraph/durable-execution

**제안**: “checkpoint + thread_id로 workflow 진행 상태를 저장하고, 중단 시 마지막 기록 상태에서 재개한다”처럼 paraphrase로 정정하세요.

### 4. n8n quote 및 외부 오케스트레이션 단정 오류

**위치**: L38

**현재**: `"AI Agent node operates as a single LLM call within n8n's own execution environment and cannot orchestrate agents running on external infrastructure"`

**문제**: [ACCEPT] 사실/인용 위반. LowCode Agency 글에서 이 문장은 확인되지 않고, 같은 글은 n8n의 AI agent node가 structured workflow 안에서 reasoning/tool loop를 수행한다고 설명합니다. 또한 LangChain·CrewAI·AutoGen을 HTTP로 호출해 결합할 수 있다고 하므로 “cannot orchestrate agents running on external infrastructure”는 출처와도 맞지 않습니다. 출처: https://www.lowcode.agency/blog/n8n-vs-agentic-ai

**제안**: 직접 quote를 제거하고 “n8n은 정의된 workflow 안에서 AI agent node를 실행하는 workflow automation 도구이며, 전용 multi-agent framework보다 구조가 제한적이지만 HTTP로 외부 agent framework를 호출할 수 있다”로 정리하세요.

### 5. Paperclip budget hard-stop 범위 과장

**위치**: L46

**현재**: `토큰·비용을 company / agent / project / goal / issue / provider / model 차원으로 추적하고 초과 시 에이전트를 자동 일시정지·큐 취소한다.`

**문제**: [ACCEPT] 사실 오류. 비용 attribution 축은 대체로 맞지만 hard-stop 집행 범위와 섞여 있습니다. `cost_events`는 `company_id`, `agent_id`, `issue_id`, `project_id`, `goal_id`, `heartbeat_run_id`, `provider`, `model` 등을 저장합니다(`packages/db/src/schema/cost_events.ts:13-27`). 그러나 budget hard-stop 정책 평가는 `company` / `agent` / `project` scope만 대상으로 합니다(`server/src/services/budgets.ts:653-663`), 시작 차단도 같은 세 scope를 확인합니다(`server/src/services/budgets.ts:716-860`). 큐/실행 취소 역시 agent/company/project scope 기준입니다(`server/src/services/heartbeat.ts:9431-9456`). 따라서 issue/goal/provider/model 차원 초과가 직접 hard-stop scope로 동작한다고 쓰면 현재 구현과 다릅니다.

**제안**: “비용은 company / agent / issue / project / goal / heartbeat / provider / model 축으로 귀속한다. hard-stop 집행 scope는 company / agent / project이며, 초과 시 해당 scope를 pause하고 관련 pending/active work를 취소한다”처럼 추적 축과 집행 scope를 분리하세요.
