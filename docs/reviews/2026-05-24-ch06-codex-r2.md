## 정합성

### 1. Route 카운트가 `<Routes>`까지 포함해 1개 많음

- **위치**: line 19
- **현재 본문 인용**: `## 2. 라우팅 표 (Route 129개)`
- **문제 카테고리**: 정합성
- **제안 수정안**: `Route 128개`로 고치거나, 카운트 산식을 명시하지 않는 `라우팅 표`로 낮추세요.
- **근거**: `rg "<Route" ui/src/App.tsx`는 `<Routes>` 컨테이너까지 잡아 129가 되지만, 실제 `<Route ...>` 컴포넌트는 128개입니다. 코드상 `<Routes>`는 `ui/src/App.tsx:279`, 실제 route들은 `ui/src/App.tsx:68-146` 및 `ui/src/App.tsx:280-334`에 있습니다.

### 2. 본문 prose에 `entity_type`이 아직 남아 있음

- **위치**: line 94
- **현재 본문 인용**: `들어온 `LiveEvent` 의 `type`/`entity_type` 으로 query key 를 invalidate`
- **문제 카테고리**: 정합성
- **제안 수정안**: `type`/`payload.entityType`으로 고치세요.
- **근거**: 실제 구현은 `ui/src/context/LiveUpdatesProvider.tsx:650`에서 `payload.entityType`을 읽고, `activity.logged` 처리는 `ui/src/context/LiveUpdatesProvider.tsx:852-853`에서 `invalidateActivityQueries(...)`로 이어집니다.

### 3. 디렉터리 트리의 `api/` 주석이 Round 1 수정 취지와 충돌함

- **위치**: line 77
- **현재 본문 인용**: `├── api/                 # fetch 래퍼 + zod 검증`
- **문제 카테고리**: 정합성
- **제안 수정안**: `# fetch 래퍼 + shared 타입 클라이언트` 또는 `# fetch 래퍼 + 일부 응답 zod 검증`으로 완화하세요.
- **근거**: 공통 fetch 래퍼는 `ui/src/api/client.ts:22-36`에서 JSON을 반환할 뿐 zod parse를 하지 않습니다. zod 검증은 auth 일부 응답에 한정되어 `ui/src/api/auth.ts:32-36`, `ui/src/api/auth.ts:125-129`에서 확인됩니다.

### 4. Vite HMR을 "동일 origin"이라고 한 설명이 현재 서버 설정과 다름

- **위치**: line 148
- **현재 본문 인용**: `**HMR 도 동일 origin** — Vite WS 가 그대로 동작한다.`
- **문제 카테고리**: 정합성
- **제안 수정안**: `HMR은 Express에 마운트된 Vite dev server 설정을 따라 별도 HMR 포트로 동작한다` 정도로 고치세요.
- **근거**: `server/src/app.ts:381-392`에서 `hmrPort = resolveViteHmrPort(opts.serverPort)`를 계산하고 Vite `server.hmr.port/clientPort`에 설정합니다. `resolveViteHmrPort`는 `server/src/app.ts:92-97`에서 기본적으로 서버 포트에 10,000을 더하므로 API/UI HTTP origin과 같은 포트가 아닙니다.
