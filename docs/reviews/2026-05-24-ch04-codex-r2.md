## F2

**위치**  
`docs/report/04-adapters-and-skills.md:95-105`

**현재 본문 인용**  
> 코드 2 가 외부 어댑터를 코어 수정 없이 *설치 → 등록 → 활성화 → 사용* 4단계로 들이는 표준 절차다 — 1단계는 CLI 한 줄, 2\~4단계는 보드 UI 안에서 마우스 클릭으로 끝난다.
>
> ```bash
> # 1) 어댑터 플러그인을 설치
> pnpm paperclipai plugin install <package-or-path>
>
> # 2) 플러그인이 ~/.paperclip/adapter-plugins.json 에 등록
> # 3) 보드 UI > Adapter manager 에서 활성화
> # 4) 새 에이전트 만들 때 adapter_type=<plugin-id>
> ```

**문제 카테고리**  
정합성

**제안 수정안**  
> 코드 2가 외부 어댑터를 코어 수정 없이 *설치 → 등록 → 사용* 흐름으로 들이는 현재 구현 경로다. 설치는 Adapter Manager 또는 `/api/adapters/install`이 담당하며, 성공하면 `~/.paperclip/adapter-plugins.json`에 adapter plugin record가 저장된다.
>
> ```text
> # 1) Board > Adapter Manager 에서 package-or-path 설치
> #    또는 POST /api/adapters/install { packageName, isLocalPath?, version? }
> # 2) 서버가 external adapter module 을 load/register 하고
> #    ~/.paperclip/adapter-plugins.json 에 record 를 저장
> # 3) 새 에이전트 만들 때 adapter_type=<adapter type>
> ```

**근거**  
`paperclipai plugin install`은 일반 Paperclip plugin 설치 CLI이고 `/api/plugins/install`로 요청을 보낸다(`cli/src/commands/client/plugin.ts:300-330`). `/api/plugins/install`은 manifest를 가진 일반 plugin을 설치·검증·DB 등록하는 경로다(`server/src/routes/plugins.ts:873-897`, `server/src/routes/plugins.ts:930-935`). 반면 외부 adapter 설치는 별도 adapter management API가 담당하며, `POST /adapters/install`에서 npm/local package를 load한 뒤 `registerWithSessionManagement(adapterModule)`을 호출하고 `addAdapterPlugin(record)`로 adapter plugin record를 저장한다(`server/src/routes/adapters.ts:222-229`, `server/src/routes/adapters.ts:315-326`). UI도 external adapter install API로 `/adapters/install`을 호출한다(`ui/src/api/adapters.ts:42-48`). adapter plugin store의 source of truth는 `~/.paperclip/adapter-plugins.json`이다(`server/src/services/adapter-plugin-store.ts:2-6`, `server/src/services/adapter-plugin-store.ts:46-50`).
