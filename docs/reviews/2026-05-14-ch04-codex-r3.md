## 종합 판정

R2의 세 수정 의도는 반영됐다. 인용 범위와 sandbox provider 명칭, `skills.managed` capability와 manifest `skills` 배열의 구분은 큰 방향에서 맞다. 다만 현재 본문에는 Cursor Cloud 반영 뒤 남은 선택 필드 설명 오류와, 새 plugin-managed skill 문장의 잘못된 타입 인용이 남아 있다.

## ACCEPT-worthy

### R3-A1. 위치: `04-adapters-and-skills.md:19`

- 현재: "`getConfigSchema`는 `acpx-local`에 한정"된다고 설명.
- 문제: 현재 registry에는 `acpx_local`뿐 아니라 `cursor_cloud`도 `getConfigSchema`를 제공한다. 라운드 1에서 `cursor_cloud`가 built-in 10번째 런타임 어댑터로 추가됐으므로, 기존 "acpx-local 한정" 문장이 이제 현재 구현과 충돌한다.
- 제안: "`getConfigSchema`는 `acpx-local`과 `cursor-cloud`가 제공한다"로 고치거나, "일부 어댑터만 제공한다"로 완화한다.
- 근거: `server/src/adapters/registry.ts:64`, `server/src/adapters/registry.ts:275`, `server/src/adapters/registry.ts:330`, `packages/adapters/cursor-cloud/src/server/index.ts:7`.

### R3-A2. 위치: `04-adapters-and-skills.md:141`

- 현재: manifest `skills` 배열의 각 항목을 `PluginManagedSkillDeclaration`이라고 하면서 근거를 `packages/plugins/sdk/src/types.ts:852-862`로 붙인다.
- 문제: `packages/plugins/sdk/src/types.ts:852-862`는 `PluginSkillsClient`의 `get/reconcile/reset` API 설명이지 `PluginManagedSkillDeclaration` 선언이나 manifest `skills` 배열 타입이 아니다. 타입 자체는 plugin SDK에서 re-export되고, manifest의 `skills` 배열은 shared validator schema에서 정의된다.
- 제안: 괄호 인용을 "`packages/shared/src/validators/plugin.ts:246-269`, `:624-628`; SDK re-export는 `packages/plugins/sdk/src/index.ts:260-273`"로 교체한다. `get/reconcile/reset` 흐름 근거로는 기존 `packages/plugins/sdk/src/types.ts:852-862`를 별도 문장에 붙이면 된다.
- 근거: `packages/shared/src/validators/plugin.ts:246-269`, `packages/shared/src/validators/plugin.ts:624-628`, `packages/plugins/sdk/src/index.ts:260-273`, `packages/plugins/sdk/src/types.ts:852-862`.
