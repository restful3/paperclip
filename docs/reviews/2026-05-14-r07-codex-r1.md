R07 본문은 pnpm의 핵심 모델과 Paperclip 모노레포의 큰 구조를 잘 잡고 있지만, Paperclip 실제 워크스페이스 글롭과 어댑터 목록에서 몇 가지 stale fact가 보입니다. 특히 `cursor-cloud` 누락과 잘못된 `--filter` 패키지명은 그대로 두면 사용자가 실행 가능한 명령을 잘못 받아들일 수 있습니다.

### 1. Paperclip 워크스페이스 글롭 누락

**위치**: L87

**현재**: `Paperclip 의 루트 pnpm-workspace.yaml 은 packages/*, packages/adapters/*, packages/plugins/*, server, ui, cli 를 워크스페이스로 묶는다`

**문제**: [ACCEPT] 사실 정확성. 실제 `pnpm-workspace.yaml` 은 위 글롭 외에도 `packages/plugins/examples/*` 를 포함하고, `!packages/plugins/sandbox-providers/**` 와 `!packages/plugins/examples/plugin-orchestration-smoke-example` 두 exclude 패턴을 둡니다. 현재 문장은 예제 플러그인 워크스페이스와 두 번째 exclude를 빠뜨립니다. 진실 소스: `pnpm-workspace.yaml:1-14`; 실측 `pnpm -r list --parseable --depth -1` 에서 `packages/plugins/examples/plugin-authoring-smoke-example` 등 예제 플러그인 4개가 워크스페이스로 잡힙니다.

**제안**: 글롭 나열을 `packages/*`, `packages/adapters/*`, `packages/plugins/*`, `packages/plugins/examples/*`, `server`, `ui`, `cli` 로 보강하고, exclude는 sandbox-provider 전체와 `plugin-orchestration-smoke-example` 두 가지라고 명시하세요.

### 2. 내장 어댑터 패키지 목록

**위치**: L87

**현재**: `claude-local·codex-local·cursor-local·gemini-local·openclaw-gateway·opencode-local·acpx-local·pi-local 등`

**문제**: [ACCEPT] 사실 정확성. Paperclip의 패키지형 내장 어댑터는 9개이며, 현재 문장에는 `cursor-cloud` 가 빠져 있습니다. R05/Ch01 합의도 9개 목록을 사용합니다. 진실 소스: `server/package.json:47-55`, `docs/report/01-architecture.md:39`, `packages/adapters/cursor-cloud/package.json:1-2`.

**제안**: 목록을 `acpx-local`, `claude-local`, `codex-local`, `cursor-cloud`, `cursor-local`, `gemini-local`, `openclaw-gateway`, `opencode-local`, `pi-local` 로 완전하게 쓰세요.

### 3. 서버 패키지 필터명

**위치**: L87

**현재**: ``pnpm --filter @paperclip/server dev``

**문제**: [ACCEPT] 사실 정확성. 실제 서버 패키지명은 `@paperclipai/server` 입니다. 현재 명령은 워크스페이스 패키지를 매칭하지 못합니다. 진실 소스: `server/package.json:1-3`; 루트 `package.json` 도 `pnpm --filter @paperclipai/server ...` 를 사용합니다(`package.json:21-26`).

**제안**: 명령을 ``pnpm --filter @paperclipai/server dev`` 로 고치세요.

### 4. overrides 위치와 Paperclip 현재 설정

**위치**: L54

**현재**: `pnpm-workspace.yaml 의 overrides ... 이 필드는 개별 package.json 이 아니라 워크스페이스 루트의 yaml 에만 둔다`

**문제**: [ACCEPT] 사실 정확성. 최신 pnpm 문서의 Settings 페이지는 `pnpm-workspace.yaml` 설정을 설명하지만, Paperclip은 `packageManager: pnpm@9.15.4` 를 쓰고 실제 override를 루트 `package.json` 의 `pnpm.overrides` 아래에 둡니다. R07은 Paperclip 구조와 연결되는 문서이므로 “yaml 에만 둔다”는 단정은 현재 레포와 충돌합니다. 진실 소스: `package.json:63-70`, `pnpm-workspace.yaml:1-14`, https://pnpm.io/settings#overrides.

**제안**: “최신 pnpm 설정 문서에서는 `pnpm-workspace.yaml` 에 둘 수 있고, Paperclip 현재 레포는 pnpm 9 계열 방식으로 루트 `package.json` 의 `pnpm.overrides` 를 사용한다”로 버전/레포 차이를 분리하세요.
