# 챕터 04 리뷰 — 라운드 1

## 종합 판정

CHANGE-IMPACT의 STALE 판정이 맞다. Ch04는 현재 구현의 어댑터 수, Cursor Cloud 추가, sandbox provider 플러그인, plugin-managed skills를 반영하지 못한다. 또한 현재 구현 계약을 설명해야 하는 자리에서 장기 SPEC의 `invoke/status/cancel` 명칭이 다시 섞여 있어 Ch03에서 합의한 "SPEC 계약 vs 현재 `execute/testEnvironment` 구현" 구분과 충돌한다.

## ACCEPT-worthy

### R1-A1. 위치: `04-adapters-and-skills.md:13`, `:15`, `:25`, `assets/fig-07-adapter-matrix.svg:72-88`

- 현재: built-in 어댑터를 "런타임 전용 9개 + 범용 2개"로 설명하고, 표 1과 그림 7 모두 `cursor_cloud` 없이 `cursor-local` 다음이 `gemini-local`이다.
- 문제: 현재 registry와 built-in type set에는 `cursor_cloud`가 포함되어 총 12개다. 런타임 전용은 10개(`acpx_local`, `claude_local`, `codex_local`, `cursor_cloud`, `cursor`, `gemini_local`, `openclaw_gateway`, `opencode_local`, `pi_local`, `hermes_local`)이고 generic은 `process`/`http` 2개다.
- 제안: 본문/캡션/표 제목을 "런타임 전용 built-in 10개 + 범용 2개"로 고치고, 표 1과 그림 7에 `cursor_cloud` 행을 추가한다. 행 설명은 "Cursor Cloud Agent / Cursor SDK 기반 managed remote agent / `cursor_cloud`" 정도로 잡으면 된다.
- 근거: `server/src/adapters/registry.ts:480-493`, `server/src/adapters/builtin-adapter-types.ts:4-17`, `packages/adapters/cursor-cloud/src/index.ts:1-33`, `ui/src/adapters/adapter-display-registry.ts:101-104`.

### R1-A2. 위치: `04-adapters-and-skills.md:61-87`, `:145`, `assets/fig-07-adapter-matrix.svg:9-19`

- 현재: 어댑터 `./server` 진입점 책임을 `invoke / status / cancel` 구현으로 쓰고, 마지막에도 SKILL/MCP가 "어댑터 계약(invoke / status / cancel)"과 직교한다고 설명한다. 그림 7도 첫 capability 열을 `invoke()`로 표시한다.
- 문제: `invoke/status/cancel`은 장기 SPEC의 최소 인터페이스 명칭이고, 현재 구현의 `ServerAdapterModule` 필수 메서드는 `execute(ctx)`와 `testEnvironment(ctx)`다. `status`/`cancel`은 현재 server adapter module의 필수 계약이 아니며, Ch03 합의와도 어긋난다.
- 제안: 코드 1의 `./server` 주석과 표 2의 책임을 "`execute` / `testEnvironment` + optional lifecycle/capability hooks"로 바꾸고, `:145`도 "현재 어댑터 실행 계약(`execute`/`testEnvironment`)과 수직 직교"로 수정한다. 그림 7의 `invoke()` 축도 `execute()` 또는 `callable execute`로 바꾸는 편이 맞다.
- 근거: `packages/adapter-utils/src/types.ts:349-352`, `packages/adapter-utils/src/types.ts:353-431`, `doc/SPEC.md:207-215`.
