[9-4] §2 표 1 #6 하드 일시정지 근거는 실제 심볼명과 역할을 더 정확히 써야 합니다.

- 진실 소스: start lock은 실제로 존재하지만 파일/심볼은 `server/src/services/agent-start-lock.ts:32` 의 `withAgentStartLock` 이고, queued run start 직렬화 및 paused 상태 게이트에 쓰입니다(`server/src/services/heartbeat.ts:6688-6693`). 예산 하드스톱의 직접 실행 차단은 `budgets.getInvocationBlock` 호출(`server/src/services/heartbeat.ts:8610-8618`)과 budget pause 취소 훅(`server/src/services/heartbeat.ts:9431-9456`)입니다. SPEC의 hard ceiling도 `doc/SPEC.md:295-300` 입니다.
- 분류: ⚠️ 표현·인용 완결성
- 권고: `agent_start_lock 서비스` 단독 근거처럼 보이지 않게, `budgetService.getInvocationBlock` / `cancelBudgetScopeWork` / `withAgentStartLock` 의 역할을 나눠 적으세요.

[9-5] §3 표 2 #3 "단일 보드"는 현재 코드의 multi-user access를 과소서술합니다.

- 본문 인용: `SPEC §1, instance_user_roles의 단일 admin 모델`
- 진실 소스: 회사 멤버십은 회사별 user principal과 role을 가집니다(`packages/db/src/schema/company_memberships.ts:4-13`), human role은 `owner/admin/operator/viewer` 입니다(`packages/shared/src/constants.ts:624-630`). instance role은 `instance_admin` 한 종류지만 singleton admin 모델은 아닙니다(`packages/db/src/schema/instance_user_roles.ts:3-14`). 다만 고급 거버넌스와 multi-member boards는 Not V1로 남아 있고(`doc/SPEC.md:489-493`), `approvalsNeeded > 1` 은 거부됩니다(`server/src/__tests__/issue-execution-policy.test.ts:98-105`).
- 분류: ❌ 사실 오류
- 권고: 약점을 "멀티 사용자 자체 부재"가 아니라 "고급 거버넌스/4-eye 결재/다중 승인 정책 부재"로 좁히세요.
