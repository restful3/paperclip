### F1

- **위치**: line 106
- **현재 본문 인용**: `첫째, 사람 보드는 회사 *전체 트리*가 아니라 *CEO 한 명*을 직접 관리한다 — CEO 아래 임원층의 결정은 CEO가 보드에 집계해 올린다.`
- **문제 카테고리**: 정합성
- **제안 수정안**: `운영 원칙으로는 CEO 중심 위임을 택할 수 있지만, 현재 보드 UI/API는 전체 org tree의 에이전트를 직접 조회·생성·수정할 수 있다. 따라서 깊은 트리는 “보드가 직접 봐야 할 표면”보다 “책임 추적 표면”을 넓힌다.`처럼 단정 완화.
- **근거**: 보드 라우팅은 `/org`, `/agents/all`, `/agents/new`, `/agents/:agentId` 등 전체 에이전트/조직 화면을 노출한다(`ui/src/App.tsx:87-96`). 서버도 보드가 회사 범위 에이전트를 관리할 수 있는지 검사하는 `assertBoardCanManageAgentsForCompany()`를 두고(`server/src/routes/agents.ts:571-580`), agent detail/update 경로에서 전체 agent id를 대상으로 update 권한을 검사한다(`server/src/routes/agents.ts:2540-2548`, `:677-685`). 따라서 "보드는 전체 트리가 아니라 CEO 한 명만 직접 관리한다"는 현재 구현 설명으로는 과하다.
