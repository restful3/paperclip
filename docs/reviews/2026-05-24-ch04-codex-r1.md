**문제 카테고리**  
정합성

**제안 수정안**  
표 제목을 다음처럼 바꾸고, `gemini-local`과 `opencode-local` 사이 또는 적절한 위치에 `grok-local` 행을 추가한다.

> **표 1. 런타임 전용 built-in 11종 + 범용 2종**

> | `grok-local` | xAI Grok Build CLI | `grok --output-format streaming-json --single …` | ● | proprietary xAI/Grok service · SuperGrok Heavy 또는 xAI API 경로 |

또는 라이선스/요금 단언을 낮추려면:

> | `grok-local` | xAI Grok Build CLI | `grok --output-format streaming-json --single …` | ● | proprietary xAI/Grok service · account/API billing |

**근거**  
현재 built-in adapter 등록은 13개이고, 그중 runtime-specific adapter는 `grokLocalAdapter`를 포함한 11개다(`server/src/adapters/registry.ts:513-525`). `grok-local`은 workspace adapter dependency에도 포함된다(`server/package.json:47-56`). 실제 Grok adapter는 `type = "grok_local"`, label `"Grok Build (local)"`, 기본 모델 `grok-build`를 선언한다(`packages/adapters/grok-local/src/index.ts:1-8`). 실행 인자는 `--output-format streaming-json`와 `--single`을 사용한다(`packages/adapters/grok-local/src/server/execute.ts:431-448`). xAI 공식 문서도 Grok Build가 터미널 coding agent이며 headless에서 `--output-format streaming-json`을 지원한다고 설명한다(https://docs.x.ai/build/overview, lines 228, 259-266). xAI launch post는 2026-05-14 기준 early beta가 SuperGrok Heavy subscribers 대상이라고 설명한다(https://x.ai/news/grok-build-cli, lines 88-98, 250-252). 현재 표는 `grok-local`이 빠져 있어 10+2=12개만 설명한다.
