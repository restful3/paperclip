# 챕터 05 리뷰 — 라운드 2

## 종합 판정

R1 ACCEPT 6건은 대부분 반영됐다. WebSocket 인증, live event type 9개, secrets 확장, recovery-actions API, org chart mount 위치, plugin-managed skills 누락은 본문에 들어왔다. 다만 수정 과정에서 대표 endpoint path와 plugin hook 카운트/서비스 파일 표현에 substantive 오류가 남았다.

## ACCEPT-worthy

### R2-A1. 위치: `05-server-api.md:157-169`

- 현재: "plugin SDK의 5가지 hook 카테고리"라고 쓰면서 코드 3에는 6개 카테고리(`plugin lifecycle`, `data namespaces`, `events / webhooks`, `UI extensions`, `adapter plugins`, `managed resources`)를 나열한다.
- 문제: 카운트가 맞지 않는다. R1에서 managed resources를 추가하면서 제목과 설명의 "5가지"가 stale해졌다.
- 제안: "6가지 hook 카테고리"로 수정한다.
- 근거: `05-server-api.md:161-169`.

### R2-A2. 위치: `05-server-api.md:167-169`

- 현재: managed resources 설명에서 `agents / projects / routines / skills`를 manifest로 선언하면 `plugin-managed-{agents,routines,skills}.ts`가 동기화한다고 쓴다.
- 문제: agents/routines/skills는 각각 `plugin-managed-agents.ts`, `plugin-managed-routines.ts`, `plugin-managed-skills.ts`가 맞지만, project managed resource는 별도 `plugin-managed-projects.ts`가 아니라 `projects.ts`의 `resolveManagedProject`가 처리한다. 현재 문장은 projects까지 `plugin-managed-*` 파일군이 처리하는 것처럼 읽힌다.
- 제안: "agents/routines/skills는 `plugin-managed-{agents,routines,skills}.ts`, projects는 `projects.ts`의 `resolveManagedProject` 경로가 회사 스코프 자원과 동기화"로 고친다.
- 근거: `server/src/services/plugin-host-services.ts:489-522`, `server/src/services/plugin-managed-agents.ts:173`, `server/src/services/plugin-managed-routines.ts:125`, `server/src/services/plugin-managed-skills.ts:156`, `server/src/services/projects.ts:533-690`.

## DEFER

- `secrets.ts` 대표 엔드포인트의 세부 구성은 충분히 현재 확장을 반영한다. 추가 endpoint를 더 넣는 것은 표 밀도 문제라 DEFER.
- 라인 범위 1~2줄 수준의 인용 정밀화는 ACCEPT로 잡지 않았다.
