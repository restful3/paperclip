R06 본문은 Drizzle/PGlite/embedded-postgres의 큰 비교축은 대체로 맞지만, Drizzle Kit 명령 묶음과 Paperclip §4의 코드 연결부에 substantive한 사실 오류가 남아 있습니다. 특히 Paperclip 테이블 수와 마이그레이션 대상 결정 경로는 이전 Ch02 합의와 현재 코드 모두에 맞춰 고쳐야 합니다.

### 1. Drizzle Kit 4개 명령 묶음

**위치**: L11, L22

**현재**: `drizzle-kit generate` · `migrate` · `push` · `studio` 를 “마이그레이션 워크플로” 4개 명령으로 제시

**문제**: [ACCEPT] 사실 정확성. 현재 Drizzle Migrations 문서의 기본 CLI 묶음은 `migrate`, `generate`, `push`, `pull` 이며, `studio` 는 스키마/데이터를 보는 별도 UI 명령입니다. 따라서 `studio` 를 migration workflow 4개 명령 안에 넣으면 문서의 명령 분류와 다릅니다. 진실 소스: https://orm.drizzle.team/docs/migrations, https://orm.drizzle.team/docs/drizzle-kit-studio.

**제안**: 본문과 표를 `drizzle-kit generate/migrate/push/pull` 로 고치고, `studio` 는 “시각적 DB 관리 UI”로 별도 문장에 분리하세요.

### 3. 마이그레이션 대상 결정 경로

**위치**: L40

**현재**: ``drizzle.config.ts` 가 `process.env.DATABASE_URL` 한 줄을 기준으로 마이그레이션 대상을 결정한다`

**문제**: [ACCEPT] 사실 정확성. `packages/db/drizzle.config.ts` 는 `drizzle-kit generate` 등 Drizzle Kit 설정에서 `process.env.DATABASE_URL` 을 읽지만, Paperclip의 실제 `pnpm db:migrate` 경로는 `packages/db/src/migrate.ts` 가 `resolveMigrationConnection()` 을 호출하고, 그 안에서 `DATABASE_URL`/`.paperclip` env/config/embedded-postgres fallback 을 순서대로 해석합니다. 즉 “마이그레이션 대상” 전체를 `drizzle.config.ts` 한 줄이 결정한다고 쓰면 현재 코드 경로와 다릅니다. 진실 소스: `packages/db/drizzle.config.ts:3-9`, `packages/db/src/migrate.ts:1-17`, `packages/db/src/runtime-config.ts:213-264`, `packages/db/src/migration-runtime.ts:183-193`.

**제안**: “`drizzle.config.ts` 는 Drizzle Kit 설정에서 `DATABASE_URL` 을 사용하고, `pnpm db:migrate` 는 `resolveMigrationConnection()` 을 통해 외부 Postgres 또는 embedded-postgres 를 선택한다”로 분리해 쓰세요.
