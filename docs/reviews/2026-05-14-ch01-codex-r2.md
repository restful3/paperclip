# 챕터 01 리뷰 — 라운드 2

## 적용된 변경 검토

라운드 1 ACCEPT 7건은 본문에 반영된 것으로 확인했다. adapter type 12개, `cursor-cloud` 포함, `execute / testEnvironment` 계약 구분, `mcp-server` 의 DB 직접 의존 제거, `detect-port` 기반 포트 설명, 인스턴스 디렉터리 축약 고지, fork-specific 포트 표현 제거, 주요 `path:line` 인용 보강 모두 방향은 맞다.

다만 표 1의 의존 열과 빌드 명령 설명에서 새로 확인되는 사실 오류가 남아 있다. 둘 다 독자가 모노레포 토폴로지를 이해하는 핵심 문장이므로 ACCEPT-worthy로 본다.

## ACCEPT-worthy

### R2-A1. 표 1 의존 열이 실제 `package.json` 의존과 아직 맞지 않음 (`01-architecture.md:26-28`)

[기준 1: 사실 정확성, 기준 5: 인용 완결성]

`ui/` 행은 의존을 `shared (런타임은 fetch)` 로만 적고 있지만, 실제 `ui/package.json` 은 9개 adapter workspace 패키지, `@paperclipai/adapter-utils`, `@paperclipai/shared`, `hermes-paperclip-adapter` 를 함께 의존한다(`ui/package.json:34-54`). 본문 54줄은 UI가 adapter `ui` entry를 가져간다고 설명하므로, 표에서도 adapter UI 패키지 의존을 숨기면 구조 설명이 어긋난다.

`cli/` 행도 `shared · server` 로 축약되어 있지만, 실제로는 adapter workspace 패키지들, `adapter-utils`, `db`, `server`, `shared` 를 함께 의존한다(`cli/package.json:44-52`).

`packages/db` 행은 `embedded-postgres` 만 적고 있지만, 실제 runtime dependencies는 `@paperclipai/shared`, `drizzle-orm`, `embedded-postgres`, `postgres` 다(`packages/db/package.json:47-50`). 최소 수정은 표의 열 이름을 "주요 의존" 또는 "주요 내부/런타임 의존"으로 명확히 하고, 위 세 행을 실제 의존과 맞추어 갱신하는 것이다.

### R2-A2. 빌드 명령 설명이 `pnpm -r build` 와 루트 `pnpm build` 를 혼동함 (`01-architecture.md:73`, `01-architecture.md:92`)

[기준 1: 사실 정확성]

본문은 "빌드는 `pnpm -r build` 로 ... `preflight:workspace-links` 단계가 우선 ..."이라고 설명하고, 표 2도 표준 명령을 `pnpm -r build` 로 적는다. 하지만 실제 preflight를 먼저 실행하는 것은 루트 `pnpm build` 이고, 그 스크립트가 `pnpm run preflight:workspace-links && pnpm -r build` 로 이어진다(`package.json:16`). `pnpm -r build` 를 직접 실행하면 이 preflight 단계는 자동으로 앞서지 않는다.

수정 방향은 둘 중 하나면 된다. 표준 명령을 `pnpm build` 로 바꾸고 "루트 스크립트가 preflight 후 `pnpm -r build` 를 호출한다"고 설명하거나, `pnpm -r build` 는 하위 빌드 단계이며 preflight 포함 표준 진입점은 `pnpm build` 라고 분리해 적으면 된다.

## DEFER

- D1. "의존이 위에서 아래로만 흐른다"는 큰 방향은 유지 가능하지만, 표 1의 실제 의존 갱신 후 다시 읽는 편이 낫다. 현재는 UI/CLI adapter 의존 누락 때문에 독자가 그림보다 단순한 구조로 오해할 수 있다.
- 한국어 조사 띄어쓰기(`표 1 에서`, `server 가`, `UI 는`)는 자연성 측면에서 정리하면 좋지만, 이번 라운드의 핵심 사실 오류보다 우선하지 않는다.
